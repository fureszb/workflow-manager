# Workflow Manager – Indítási útmutató (szűz Ubuntu 24 LTS)

## Mi ez a projekt?
Háromrétegű webalkalmazás:
- **Frontend** – React + Vite (böngészőben fut) → http://localhost:5173
- **Backend** – Python FastAPI → http://localhost:8000
- **Adatbázis** – MySQL (háttérben fut, 3306-os port)

---

## 1. LÉPÉS – Szükséges programok telepítése

### Rendszer frissítése
```
sudo apt update && sudo apt upgrade -y
```

### Node.js 20 telepítése (frontendhez kell)
```
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
```

Ellenőrzés:
```
node --version
npm --version
```

### Python 3.11+ telepítése (backendhez kell)
```
sudo apt install -y python3 python3-pip python3-venv
```

Ellenőrzés:
```
python3 --version
```

### MySQL telepítése (adatbázis)
```
sudo apt install -y mysql-server
sudo systemctl start mysql
sudo systemctl enable mysql
```

---

## 2. LÉPÉS – MySQL adatbázis beállítása

Lépj be MySQL-be:
```
sudo mysql
```

A `mysql>` promptban másold be ezt (egyben):
```sql
CREATE DATABASE workflow_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'testuser'@'localhost' IDENTIFIED BY '1122';
GRANT ALL PRIVILEGES ON workflow_manager.* TO 'testuser'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

---

## 3. LÉPÉS – Backend indítása (1. terminál)

```
# Menj a backend mappába (a projekt gyökeréből)
cd backend

# Virtuális Python környezet létrehozása (csak ELSŐ alkalommal)
python3 -m venv venv

# Virtuális környezet aktiválása (minden indításkor kell!)
source venv/bin/activate

# Python csomagok telepítése (csak ELSŐ alkalommal)
pip install -r requirements.txt

# Adatbázis táblák létrehozása (csak ELSŐ alkalommal, vagy ha új migráció van)
alembic upgrade head

# Teszt adatok betöltése (csak ELSŐ alkalommal, opcionális)
python seed_data.py

# Backend indítása
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Ha sikeres, ezt fogod látni:
```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

FONTOS: NE ZÁRD BE ezt a terminált! Hagyd futni!

---

## 4. LÉPÉS – Frontend indítása (2. terminál)

Nyiss egy ÚJ terminált (Ctrl+Alt+T), majd:

```
# Menj a frontend mappába (a projekt gyökeréből)
cd frontend

# Node csomagok telepítése (csak ELSŐ alkalommal)
npm install

# Frontend indítása
npm run dev
```

Ha sikeres, ezt fogod látni:
```
  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

FONTOS: NE ZÁRD BE ezt a terminált sem!

---

## 5. LÉPÉS – Megnyitás böngészőben

Nyisd meg a böngészőt és írd be:
```
http://localhost:5173
```

## Összefoglaló – Mi hol fut?

| Terminál    | Szolgáltatás         | Port | Parancs                                          |
|-------------|----------------------|------|--------------------------------------------------|
| 1. terminál | Backend (FastAPI)    | 8000 | uvicorn app.main:app --reload --host 0.0.0.0 --port 8000 |
| 2. terminál | Frontend (Vite)      | 5173 | npm run dev                                      |
| Háttérben   | MySQL adatbázis      | 3306 | automatikusan fut (systemd)                      |

## Hibák és megoldásuk

### "Access denied for user 'testuser'"
A MySQL felhasználó nincs jól beállítva. Ismételd meg a 2. lépést.

### "Connection refused" MySQL hiba
A MySQL nem fut:
```
sudo systemctl start mysql
sudo systemctl status mysql
```

### "Module not found" vagy "alembic: command not found"
Nem aktiváltad a virtuális környezetet:
```
source venv/bin/activate
```

### "npm: command not found"
A Node.js nincs telepítve. Ismételd meg az 1. lépés Node.js részét.

### "Error generating ideas: 500" (AI funkció)
Az AI modell nincs beállítva. Két lehetőség:
1. Helyi Ollama modell telepítése:
   ```
   curl -fsSL https://ollama.com/install.sh | sh
   ollama pull ajindal/llama3.1-storm:8b-q4_k_m
   ```
2. Felhő AI (OpenRouter): állítsd be az API kulcsot a Settings oldalon.

### Frontend frissítés nem jelenik meg
Nyomd meg Ctrl+C a frontend terminálban, majd indítsd újra:
```
npm run dev
```
Ezután a böngészőben: Ctrl+F5 (hard refresh).
